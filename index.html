About

I build and lead teams working on distributed data infrastructure. At Alluxio, I lead metadata management, performance benchmarking, and end-to-end optimizations as a Tech Lead Manager and Project Management Committee member.

My path through systems has followed a thread from low-level up: I started with OS research on Akaros, a many-core operating system at UC Berkeley's AMPLab, then built Ratchet, a system for automated online schema evolution in PostgreSQL, for my PhD under Eric Brewer. That work on making live database changes safe and automatic led me to the data infrastructure world — Alluxio, Presto, Trino, and Apache Iceberg.

Broadly interested in all things systems — distributed systems, database internals, operating systems — especially the performance side.

Featured Work

Ratchet — Automated Online Schema Evolution

My PhD dissertation system that lets database administrators perform efficient, zero-downtime schema evolution on PostgreSQL. Ratchet uses fine-grained locking and a delayed-copy strategy to migrate schemas without taking the database offline. Validated by automatically replaying five years of Wikimedia schema changes.
